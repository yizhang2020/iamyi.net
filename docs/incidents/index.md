# Incidents and trends

Timely notes: public breach or attack write-ups, vendor disclosures, regulatory changes, and “what changed this quarter” for a technology you follow.

Keep each analysis focused: **what happened**, **who is affected**, **what defenders should verify**, and **what you are still uncertain about**.

## How to use this section

1. Copy `_template-incident-note.md` in this folder to a new dated filename (for example `2026-example-vendor-breach.md`), fill it in, and add the new file to `mkdocs.yml` under **Incidents & trends** (or a subsection you introduce later).
2. Link it from this page under the right year or theme.
3. When the story is no longer “live,” you can leave the page as historical context or add a short “retrospective” subsection.
